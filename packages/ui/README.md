# `@auto-deck/ui`

Shared UI components, hooks, and utilities built on [shadcn] and [base-ui].
Everything here is domain-free: code in this package never imports the Auto Deck domain packages.

[shadcn]: https://ui.shadcn.com
[base-ui]: https://base-ui.com

## Installation

From the repo root:

```sh
pnpm add @auto-deck/ui --workspace --filter <package>
```

Or add it to the `dependencies` of a workspace package's `package.json`:

```jsonc
{
  "dependencies": {
    "@auto-deck/ui": "workspace:*"
  }
}
```

## Exports

Sources are exported directly with no build step; the consuming app's bundler transpiles the `.tsx` files.

| Subpath          | Contents                                        |
|:-----------------|:------------------------------------------------|
| `./globals.css`  | The Tailwind theme and design tokens            |
| `./components/*` | Components (e.g., `Button`, `Input`, `ThemeProvider`) |
| `./hooks/*`      | Hooks (e.g., `useLocalStorage`)                 |
| `./lib/*`        | Utilities (e.g., `cn`)                          |

```ts
import { Button } from '@auto-deck/ui/components/button';
import { cn } from '@auto-deck/ui/lib/utils';
```

## Adding components

Run the shadcn CLI from the consuming app; its `components.json` aliases route primitives into this package automatically.

```sh
pnpm -C apps/playground exec shadcn add <component>
```

## Styling

The package owns the Tailwind theme in `src/styles/globals.css`, following the [shadcn monorepo layout][monorepo].
A consuming app imports it once at its entry point instead of shipping its own stylesheet:

```ts
// apps/playground/src/main.tsx
import '@auto-deck/ui/globals.css';
```

The stylesheet declares `@source "../"` so Tailwind scans this package's sources, which sit outside the consuming app's root; the app's own sources are picked up by automatic detection.

[monorepo]: https://ui.shadcn.com/docs/monorepo

## Testing

Hand-written code in this package counts toward coverage and is expected to be tested.
Sources generated by the shadcn CLI are not, and stay out of coverage via the `coverage.exclude` lists in this package's `vitest.config.ts` and the repo root's; extend both when the CLI adds files.
